# password generator & strength checker
# password-tool

Небольшой учебный CLI‑инструмент на Python для генерации паролей и оценки их сложности.  
Проект предназначен для обучения основам безопасной генерации паролей и не заменяет профессиональные менеджеры паролей или корпоративные решения. [web:3][web:19]

## Возможности

- Генерация случайных паролей с настраиваемой длиной и наборами символов.
- Оценка силы существующего пароля (числовой скоринг и словесная категория).
- Простые рекомендации по улучшению пароля.
- Примитивная проверка “похожести на словарное слово”.

## Запуск

### Генерация пароля
Параметры:

- `--length` — длина пароля (int, по умолчанию 16).
- `--lower` — использовать строчные буквы (`True`/`False`, по умолчанию `True`).
- `--upper` — использовать заглавные буквы (`True`/`False`, по умолчанию `True`).
- `--digits` — использовать цифры (`True`/`False`, по умолчанию `True`).
- `--symbols` — использовать спецсимволы (`True`/`False`, по умолчанию `True`).

Примеры:

Базовая генерация (16 символов со всеми типами)
python main.py generate

Только буквы и цифры, длина 24
python main.py generate --length 24 --symbols False

Только буквы, длина 12
python main.py generate --length 12 --digits False --symbols False
### Проверка пароля

python main.py check --password "My$ecret123!"

text

Если аргумент `--password` не указан, пароль запрашивается через скрытый ввод: [web:10][web:7]

python main.py check
Enter password to check:

text

Вывод содержит:

- числовой скоринг от 0 до 100;
- категорию: `very weak`, `weak`, `medium`, `strong`, `very strong`;
- флаг “dictionary-like”;
- список рекомендаций (если есть).

## Как считается сила пароля

Оценка носит демонстрационный характер и реализует простую эвристику:

- чем длиннее пароль, тем выше базовый балл;
- добавляются баллы за использование разных типов символов (строчные, заглавные, цифры, спецсимволы);
- вычитаются баллы за длинные повторяющиеся последовательности символов;
- вычитаются баллы за очевидные шаблоны (только буквы, только цифры и т.п.);
- если пароль похож на очевидное словарное слово или распространённый шаблон (`password`, `qwerty`, `123456` и т.п.), применяется сильный штраф.

В результате формируется итоговый балл (0–100) и категория уровня сложности.  
Модель оценки упрощённая и предназначена только для учебных целей. [web:19]

## Почему используется `secrets`, а не `random`

Для генерации паролей используется модуль `secrets`, так как он предоставляет криптографически стойкие случайные значения, подходящие для паролей, токенов и других чувствительных данных. [web:3][web:20]  
Обычный модуль `random` ориентирован на задачи моделирования и статистики, а его выход может быть предсказуем при знании внутреннего состояния генератора, что делает его неподходящим для задач безопасности.
